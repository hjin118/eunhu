# P학원 회원 관리 시스템 PRD

## 1. 제품 개요

### 1.1 시스템 개요
P학원 운영 시스템은 학원의 전반적인 회원 관리 업무를 지원하는 브라우저 기반 웹 애플리케이션입니다. 회원 등록부터 출결, 수강 관리, 결제, 상담까지 원스톱으로 관리할 수 있으며, 서버 없이 브라우저만으로 운영이 가능합니다.

### 1.2 목표
- 학원 관리 업무의 효율화 및 디지털화
- 회원 정보와 운영 데이터의 체계적 관리
- 모바일/데스크톱 환경 모두 지원
- 초기 설치 비용 없는 즉시 사용 가능한 솔루션 제공

### 1.3 타겟 사용자
- 학원 원장/관리자
- 강사
- 사무 담당자

## 2. 사용자 시나리오

### 2.1 신규 회원 등록 시나리오
1. 학원 상담원이 신규 학생 상담 진행
2. 학생 기본 정보(이름, 연락처, 학교, 학년) 입력
3. 적합한 수강반 선택 및 배정
4. 결제 등록 및 초기 상담 내용 기록
5. 회원 등록 완료 후 출결 관리 시작

### 2.2 일일 운영 시나리오
1. 관리자가 당일 수업 전날 출결 체크
2. 수업 후 진도 내용 기록
3. 당일 결제 내역 등록
4. 상담이 필요한 학생 정보 확인 및 상담 기록

### 2.3 월말 정산 시나리오
1. 월간 출결 통계 확인
2. 미납 결제 내역 확인
3. 다음 달 수강 반 재구성

## 3. 핵심 기능

### 3.1 회원 관리
**기능 개요:** 회원 정보의 등록, 조회, 수정, 삭제

**상세 기능:**
- 신규 회원 등록
  - 이름 (필수)
  - 연락처 (필수)
  - 학교 (선택)
  - 학년 (1-6, 선택)
  - 등록일 자동 기록
  
- 회원 목록 조회
  - 전체 회원 리스트 표시
  - 기본 정보 표이블로 정리
  
- 회원 정보 수정
  - 각 필드별 정보 수정 가능
  
- 회원 삭제
  - 삭제 전 확인 팝업
  - 연관 데이터 확인 로직 필요

### 3.2 출결 관리
**기능 개요:** 회원별 출결 상태 기록 및 조회

**상세 기능:**
- 출결 기록
  - 날짜 선택 (기본값: 오늘)
  - 회원 선택
  - 출결 상태 선택 (출석/결석/지각)
  
- 출결 조회
  - 기록된 출결 내역 리스트 표시
  - 날짜 역순 정렬
  - 상태별 색상 구분 (녹색/빨간색/노란색)

### 3.3 수강 반 관리
**기능 개요:** 반 생성 및 회원 배정 관리

**상세 기능:**
- 반 생성
  - 반 이름 (필수)
  - 과목 (필수)
  - 담당 강사 (필수)
  - 요일/시간 (선택)
  
- 회원 배정
  - 반별 회원 추가 기능
  - 중복 배정 방지
  
- 반 정보 조회
  - 카드 형태로 반 정보 표시
  - 구성원 목록 표시

### 3.4 수업 진도 관리
**기능 개요:** 반별 수업 진도 기록

**상세 기능:**
- 진도 기록
  - 반 선택 (필수)
  - 진도 내용 입력 (필수)
  - 날짜 선택 (기본값: 오늘)
  
- 진도 조회
  - 최신 진도 순으로 표시
  - 반 이름과 날짜 정보 함께 표시

### 3.5 결제 관리
**기능 개요:** 회원별 결제 내역 등록 및 조회

**상세 기능:**
- 결제 등록
  - 회원 선택 (필수)
  - 금액 입력 (필수)
  - 결제 유형 선택 (월 회비/교재비/기타)
  - 날짜 선택 (기본값: 오늘)
  - 상태 자동 설정 (완료)
  
- 결제 조회
  - 전체 결제 내역 표이블
  - 상태별 색상 구분
  - 금액 천 단위 콤마 표시

### 3.6 상담 기록 저장 (새로 추가)
**기능 개요:** 학생 상담 내용 기록 및 조회

**상세 기능:**
- 상담 기록
  - 회원 선택 (필수)
  - 상담 날짜 선택 (기본값: 오늘)
  - 상담 유형 선택 (입학 상담/진학 상담/성적 상담/행동 상담/기타)
  - 상담 내용 입력 (필수)
  - 후속 조치 사항 입력 (선택)
  - 다음 상담 예정일 (선택)
  
- 상담 조회
  - 회원별 상담 기록 표시
  - 최신 기록 우선 정렬
  - 상담 유형 태그 표시
  
- 상담 기록 삭제
  - 삭제 전 확인 팝업

## 4. 비기능 요구사항

### 4.1 성능 요구사항
- 페이지 로드 시간: 3초 이내
- 데이터 저장: 즉시 완료
- 조회 응답: 1초 이내

### 4.2 호환성 요구사항
- 브라우저: Chrome, Edge, Firefox, Safari (최신 버전)
- 운영체제: Windows, macOS, Linux, iOS, Android
- 화면: 모바일 (320px 이상), 태블릿, 데스크톱

### 4.3 데이터 저장 요구사항
- localStorage 사용
- 데이터 만료 없음
- 브라우저별 독립 저장

### 4.4 보안 요구사항
- 개인정보 암호화 필요 시 향후 추가
- 입력 값 유효성 검사
- XSS 방지 기본 처리

### 4.5 사용성 요구사항
- 직관적인 UI/UX
- 한글 기반 인터페이스
- 터치/클릭 모두 지원
- 필수 입력 항목 표시

## 5. 데이터 구조 설계

### 5.1 회원 (members)
```javascript
{
  id: number,           // 고유 ID
  name: string,         // 이름 (필수)
  phone: string,        // 연락처 (필수)
  school: string,       // 학교 (선택)
  grade: number,        // 학년 (1-6, 선택)
  registeredDate: string, // 등록일 (YYYY-MM-DD)
}
```

### 5.2 출결 (attendance)
```javascript
{
  id: number,           // 고유 ID
  date: string,         // 날짜 (YYYY-MM-DD)
  memberId: number,     // 회원 ID
  status: string,       // 상태 (present/absent/late)
}
```

### 5.3 수강반 (classes)
```javascript
{
  id: number,           // 고유 ID
  name: string,         // 반 이름 (필수)
  subject: string,      // 과목 (필수)
  teacher: string,      // 담당 강사 (필수)
  schedule: string,     // 요일/시간 (선택)
  members: number[],    // 회원 ID 목록
}
```

### 5.4 수업 진도 (progress)
```javascript
{
  id: number,           // 고유 ID
  classId: number,      // 반 ID
  topic: string,        // 진도 내용
  date: string,         // 날짜 (YYYY-MM-DD)
}
```

### 5.5 결제 (payments)
```javascript
{
  id: number,           // 고유 ID
  memberId: number,     // 회원 ID
  amount: number,       // 금액
  type: string,         // 유형 (monthly/material/other)
  date: string,         // 날짜 (YYYY-MM-DD)
  status: string,       // 상태 (paid/pending)
}
```

### 5.6 상담 기록 (counseling) - 새로 추가
```javascript
{
  id: number,           // 고유 ID
  memberId: number,     // 회원 ID (필수)
  date: string,         // 상담 날짜 (YYYY-MM-DD)
  type: string,         // 상담 유형 (admission/career/grade/behavior/other)
  content: string,      // 상담 내용 (필수)
  followUp: string,     // 후속 조치 (선택)
  nextDate: string,     // 다음 상담 예정일 (선택, YYYY-MM-DD)
}
```

### 5.7 데이터 저장 키
- `eunhu_members`: 회원 목록
- `eunhu_attendance`: 출결 기록
- `eunhu_classes`: 수강반 목록
- `eunhu_progress`: 수업 진도
- `eunhu_payments`: 결제 내역
- `eunhu_counseling`: 상담 기록

## 7. 확장 가능성

### 7.1 단기 확장 (3-6개월)
- **출석 통계 대시보드**
  - 월간/주간 출석률 차트
  - 학생별 출결 패턴 분석
  
- **결제 알림 기능**
  - 미납 결제 목록 표시
  - 결제 기념일 알림
  
- **엑셀 내보내기**
  - 회원 목록, 출결, 결제 내역 CSV로 다운로드

### 7.2 중기 확장 (6-12개월)
- **성적 관리**
  - 시험 점수 입력
  - 성적 변화 그래프
  - 성적 분석 리포트
  
- **일정 관리**
  - 시험 일정 캘린더
  - 행사 일정 관리
  
- **SMS/이메일 발송**
  - 결제 안내 자동 발송
  - 출석 미달 알림

### 7.3 장기 확장 (1년 이상)
- **서버 백엔드 통합**
  - 다중 사용자 동시 접속
  - 권한 관리 (원장/강사/사무)
  - 클라우드 데이터 저장
  
- **모바일 앱**
  - 네이티브 iOS/Android 앱
  - 푸시 알림
  - 오프라인 동작 지원
  
- **분석 리포트**
  - 재무 리포트
  - 학생 성장 리포트
  - 학원 운영 지표

### 7.4 기술적 확장
- **PWA로 변환**
  - 오프라인 동작 지원
  - 설치 가능한 앱
  
- **다국어 지원**
  - 영어, 중국어 등 다국어 인터페이스
  
- **데이터 백업/복원**
  - JSON 파일로 데이터 내보내기
  - 데이터 임포트 기능
  - 클라우드 백업 연동
